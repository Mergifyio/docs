---
title: "Auto-retry rules are now scoped per repository"
date: 2026-06-02
tags:
  - CI Insights
description: "Auto-retry rules are now managed per repository instead of organization-wide, with a repository selector at the top of the page."
---

Auto-retry rules now belong to a single repository instead of applying organization-wide. Pick a repository from the selector at the top of the auto-retry page to view, create, edit, or delete rules scoped to it. The `repository` condition no longer needs to be added to each rule — the scope is set by the selector itself.
